## üéØ BRAND GUIDELINES

### **Do's**

‚úÖ **Do:**
- Use confident, helpful tone
- Show real results (screenshots, metrics)
- Be transparent about pricing and limits
- Focus on developer pain points
- Use code examples and technical accuracy
- Emphasize speed and automation
- Show personality (not corporate)

### **Don'ts**

‚ùå **Don't:**
- Use corporate jargon
- Make unsubstantiated claims
- Hide pricing or limits
- Talk down to developers
- Use generic stock photos
- Be boring or safe
- Copy competitors

### **Brand Voice Checklist**

Before publishing any content, ask:
- [ ] Does this sound confident but not arrogant?
- [ ] Is this helpful and practical?
- [ ] Does this speak developer language?
- [ ] Is this transparent and honest?
- [ ] Is this action-oriented?
- [ ] Does this align with brand story?
