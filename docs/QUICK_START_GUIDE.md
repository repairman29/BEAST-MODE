## âš¡ Quick Start (VS Code)

### **1. Configure**

Open VS Code settings (`Ctrl+,` or `Cmd+,`) and search for "BEAST MODE":

```json
{
  "beastMode.apiUrl": "https://beast-mode.dev",
  "beastMode.enableSuggestions": true,
  "beastMode.enableQualityHints": true
}
```

### **2. Open a Project**

Open any JavaScript/TypeScript/Python project in VS Code.

### **3. Start Coding**

**Real-Time Suggestions:**
- Just start typing - suggestions appear automatically!
- Press `Tab` to accept a suggestion
- Press `Ctrl+Shift+B` (or `Cmd+Shift+B` on Mac) for more suggestions

**Quality Hints:**
- Quality hints appear automatically as you code
- Look for colored underlines and hover for details

**Chat:**
- Press `Ctrl+Shift+C` (or `Cmd+Shift+C` on Mac)
- Ask questions like:
  - "How do I add authentication?"
  - "Generate tests for this file"
  - "Refactor this code"

## ðŸŽ¯ Common Tasks

### **Analyze Code Quality**

1. Open any code file
2. Press `Ctrl+Shift+P` (or `Cmd+Shift+P` on Mac)
3. Type "BEAST MODE: Analyze Quality"
4. View quality score and recommendations

### **Generate Tests**

1. Open a file you want to test
2. Press `Ctrl+Shift+P`
3. Type "BEAST MODE: Generate Tests"
4. Tests are generated automatically!

### **Refactor Code**

1. Open a file
2. Press `Ctrl+Shift+P`
3. Type "BEAST MODE: Refactor"
4. Select a refactoring opportunity
5. Code is refactored automatically!

### **Index Codebase**

1. Open a workspace
2. Press `Ctrl+Shift+P`
3. Type "BEAST MODE: Index Codebase"
4. Wait for indexing to complete
5. Use fast search and context features

## ðŸ†˜ Troubleshooting

### **Suggestions Not Appearing**

1. Check that suggestions are enabled in settings
2. Make sure you're in a supported language
3. Try indexing your codebase
4. Check API URL is correct

### **Chat Not Working**

1. Check API URL in settings
2. Make sure you're connected to the internet
3. Try refreshing VS Code
4. Check browser console for errors

### **Quality Analysis Failing**

1. Make sure file is saved
2. Check file is in a supported language
3. Verify API connection
4. Check error messages in output panel
