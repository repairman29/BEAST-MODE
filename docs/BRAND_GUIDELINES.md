- **Short sentences.** Clear. Direct.
- **Active voice.** "We fix code" not "Code is fixed by us."
- **Technical accuracy.** Use correct terminology.
- **No jargon.** Avoid corporate speak.
- **Show, don't tell.** Use examples, not claims.

## üé® Visual Style Principles

### **Design Principles**

1. **Bold but clean**
   - Strong colors, minimal clutter
   - White space is your friend
   - Every element has a purpose

2. **Developer-focused**
   - Code examples, technical accuracy
   - Real screenshots, not stock photos
   - Show actual product, not concepts

3. **Action-oriented**
   - CTAs prominent and clear
   - Clear visual hierarchy
   - Guide the eye to actions

4. **Transparent**
   - Real metrics, not inflated numbers
   - Honest about limitations
   - Show actual results

5. **Modern**
   - Contemporary design
   - Not corporate or stuffy
   - Fresh and approachable

### **Do's and Don'ts**

**‚úÖ Do:**
- Use brand colors consistently
- Show real code examples
- Use actual screenshots
- Be honest about features
- Keep design clean and focused

**‚ùå Don't:**
- Use generic stock photos
- Mix too many colors
- Clutter the design
- Make unsubstantiated claims
- Copy competitors' style
